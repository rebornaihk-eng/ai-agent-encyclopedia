🔄 RAG Pipeline

📚 Source: LlamaIndex Docs, LangChain Docs

Retrieval-Augmented Generation (RAG) combines information retrieval with LLM generation to produce accurate, grounded responses based on your own data.

📐 Architecture Overview

Ingestion Pipeline

📄 Documents → Chunker → Embeddings → Vector DB

Query Pipeline

Query → Embed → Search → Context → LLM → Response

🧩 Core Components

1. Document Loaders

Load documents from various sources (PDF, Markdown, HTML, databases, APIs).

from llama_index.core import SimpleDirectoryReader

documents = SimpleDirectoryReader("./data").load_data()

2. Text Splitters / Chunkers

Split documents into smaller chunks for efficient retrieval.

Best Practice: Chunk size 512-1024 tokens with 10-20% overlap to maintain context. 
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=1000,
    chunk_overlap=200
)
chunks = splitter.split_documents(documents)

3. Embedding Models

Convert text chunks into vector representations.

5. Retrieval Strategies

  • Semantic Search - Cosine similarity between query and chunk embeddings
  • Hybrid Search - Combine semantic + keyword (BM25) search
  • Reranking - Use cross-encoder to reorder results for relevance
  • Multi-Query - Generate multiple queries to improve recall

📝 Complete Example (LlamaIndex)

from llama_index.core import VectorStoreIndex, SimpleDirectoryReader

# 1. Load documents
documents = SimpleDirectoryReader("./data").load_data()

# 2. Create index (auto chunks + embeds + stores)
index = VectorStoreIndex.from_documents(documents)

# 3. Create query engine
query_engine = index.as_query_engine(
    similarity_top_k=3,  # Retrieve top 3 chunks
    response_mode="compact"
)

# 4. Query
response = query_engine.query("What is the main topic?")
print(response)

⚡ Advanced Patterns

Hybrid RAG (Semantic + Keyword)

from llama_index.core.retrievers import VectorIndexRetriever
from llama_index.retrievers.bm25 import BM25Retriever
from llama_index.core.retrievers import QueryFusionRetriever

vector_retriever = VectorIndexRetriever(index=index, similarity_top_k=5)
bm25_retriever = BM25Retriever.from_defaults(nodes=nodes, similarity_top_k=5)

# Combine both
fusion_retriever = QueryFusionRetriever(
    retrievers=[vector_retriever, bm25_retriever],
    similarity_top_k=5
)

RAG with Reranking

from llama_index.postprocessor.cohere_rerank import CohereRerank

query_engine = index.as_query_engine(
    similarity_top_k=10  # Get more candidates
)

# Rerank to top 3
query_engine = query_engine.with_retriever(
    reranker=CohereRerank(top_n=3)
)

🎯 When to Use RAG

  • ✅ Q&A over your own documents
  • ✅ Knowledge base chatbots
  • ✅ Document search with summaries
  • ✅ Enterprise search
  • ✅ Research paper analysis

⚠️ Common Pitfalls

  1. Chunk size too large - Dilutes relevance signals
  2. No chunk overlap - Loses context at boundaries
  3. Retrieving too many chunks - Exceeds context window, adds noise
  4. Not reranking - Semantic search alone isn't perfect
  5. Ignoring metadata - Can filter by date, source, etc.
