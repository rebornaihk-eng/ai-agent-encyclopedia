🔁 ReAct Loop

📄 Source: Yao et al., 2022 (arXiv:2210.03629)

The ReAct (Reasoning + Acting) pattern enables agents to interleave thinking with action, creating a more transparent and controllable reasoning process.

📐 Official Architecture (from Paper)

Source: ReAct Paper (Yao et al., 2022)

🔄 ReAct Loop

👤 User Query
↓
💭 Thought
"I need to search for X"
↓
⚡ Action
search("X")
↓
👁️ Observation
"Found result..."
↓
Complete?
No → Loop back
↓ Yes
📤 Final Answer

🧠 How It Works

The ReAct pattern alternates between reasoning traces (Thought) and task-specific actions (Action), allowing the agent to:

  • Plan ahead before acting
  • Learn from observations
  • Explain its reasoning process
  • Handle complex multi-step tasks

📝 Example Trace

Query: "What is the population of the country where the Eiffel Tower is located?"

Thought 1: The Eiffel Tower is in Paris, France. I need to find France's population.
Action 1: search("France population 2024")
Observation 1: France has a population of approximately 68 million people as of 2024.
Thought 2: I have the information needed to answer.
Action 2: finish("France has approximately 68 million people")

Final Answer: France has approximately 68 million people as of 2024.

⚡ LangChain Implementation

from langchain.agents import initialize_agent, Tool
from langchain_openai import ChatOpenAI

# Define tools
tools = [
    Tool(
        name="Search",
        func=search_tool,
        description="Search for information on the web"
    ),
    Tool(
        name="Calculator",
        func=calculator,
        description="Perform mathematical calculations"
    )
]

# Create ReAct agent
llm = ChatOpenAI(model="gpt-4", temperature=0)
agent = initialize_agent(
    tools,
    llm,
    agent="zero-shot-react-description",
    verbose=True  # Shows thought process
)

# Run
result = agent.run("What is 25% of the population of Japan?")

🎯 When to Use ReAct

Use CaseWhy ReAct Works
Multi-step reasoningBreaks down complex tasks
Tool use requiredExplicit action selection
Explainability neededVisible thought process
Error recoveryCan observe and adjust

⚠️ Common Pitfalls

  1. Too many iterations - Set max_iterations to prevent infinite loops
  2. Poor tool descriptions - Agent can't select the right tool
  3. No early stopping - Agent keeps looping even when done
  4. Observation too long - Truncate tool outputs to avoid context overflow
Best Practice: Always set max_iterations=10 and use handle_parsing_errors=True to gracefully handle malformed actions.
