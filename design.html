🧩 10 Design Patterns

Proven patterns for building AI agents

🔄 ReAct (Reason + Act)

The foundational pattern: Think → Act → Observe → Repeat. Agent reasons step-by-step.

Thought: I need weather info
Action: search("weather HK")
Observation: 25°C, sunny
Thought: I have the answer
Answer: It's 25°C and sunny

🔗 Chain-of-Thought

Break complex problems into reasoning steps before answering. Improves accuracy.

Step 1: Understand the problem
Step 2: Identify key information
Step 3: Apply relevant knowledge
Step 4: Verify the answer
Final: Answer with confidence

👥 Multi-Agent Collaboration

Multiple specialized agents work together. Each has a role, goal, and backstory.

Researcher → finds information
Writer → creates content
Editor → reviews quality
Reviewer → final approval

📊 Hierarchical Agent

Manager delegates to workers. Good for complex projects requiring coordination.

Manager Agent
├── Worker 1 (Research)
├── Worker 2 (Development)
├── Worker 3 (Testing)
└── Worker 4 (Documentation)

👤 Human-in-the-Loop

Agent pauses for human approval before critical actions. Ensures safety.

Agent: "I found 3 options"
Agent: "Which should I use?"
Human: "Option 2"
Agent: "Proceeding with Option 2"

📚 RAG + Agent Hybrid

Combine retrieval with agent reasoning. Best for knowledge-intensive tasks.

Query → Retrieve docs → 
Agent reasons over docs → 
Generate answer with citations

🔧 Tool-Using Agent

Agent has access to tools (APIs, databases, calculators) and decides when to use them.

Agent: "I need to calculate"
Tool: calculator(123 * 456)
Result: 56088
Agent: Uses result in answer

🧠 Memory-Augmented

Agent maintains conversation history, preferences, and learned facts across sessions.

Short-term: Recent messages
Long-term: User preferences
Entity: Facts about people
Episodic: Past interactions

🔍 Self-Reflecting

Agent reviews its own output, identifies errors, and improves before responding.

Generate draft →
Critique own work →
Identify weaknesses →
Revise and improve →
Final output

📋 Planner-Executor

Separate planning (what to do) from execution (how to do it).

Planner:
1. Research topic
2. Write outline
3. Draft content
4. Review

Executor: Execute each step

📝 Code Templates

langchain_rag.py 
# LangChain RAG Template
from langchain_openai import ChatOpenAI, OpenAIEmbeddings
from langchain_community.vectorstores import Chroma
from langchain.chains import RetrievalQA
from langchain_community.document_loaders import PyPDFLoader

# 1. Load documents
loader = PyPDFLoader("docs.pdf")
docs = loader.load()

# 2. Create vector store
embeddings = OpenAIEmbeddings()
db = Chroma.from_documents(docs, embeddings)

# 3. Create RAG chain
llm = ChatOpenAI(model="gpt-4o")
qa = RetrievalQA.from_chain_type(
    llm,
    retriever=db.as_retriever(k=4)
)

# 4. Query
result = qa.invoke({"query": "What is the main topic?"})
crewai_team.py 
# CrewAI Multi-Agent Template
from crewai import Agent, Task, Crew, Process

# Define agents
researcher = Agent(
    role="Researcher",
    goal="Find comprehensive information",
    backstory="Expert at finding and synthesizing data"
)

writer = Agent(
    role="Writer",
    goal="Create engaging content",
    backstory="Award-winning technical writer"
)

# Define tasks
research = Task(
    description="Research AI agents",
    agent=researcher
)

article = Task(
    description="Write article",
    agent=writer,
    context=[research]
)

# Run crew
crew = Crew(agents=[researcher, writer], tasks=[research, article])
result = crew.kickoff()
langgraph_flow.py 
# LangGraph State Machine Template
from langgraph.graph import StateGraph, END
from typing import TypedDict

class State(TypedDict):
    messages: list
    next: str

def router(state: State) -> str:
    return "tool_agent"

def tool_agent(state: State) -> State:
    return {"messages": ["Result"]}

# Build graph
graph = StateGraph(State)
graph.add_node("router", router)
graph.add_node("tool_agent", tool_agent)

graph.set_entry_point("router")
graph.add_conditional_edges("router", router, {
    "tool_agent": "tool_agent"
})
graph.add_edge("tool_agent", END)

app = graph.compile()
